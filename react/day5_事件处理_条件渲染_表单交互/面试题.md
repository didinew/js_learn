1️⃣ React 的事件绑定和原生 JS 有什么区别？

区别点：

特性
原生 JS
React
事件名称
小写，如 onclick
驼峰，如 onClick
绑定方式
element.addEventListener('click', handler)
直接在 JSX 写 <button onClick={handleClick}>
this 指向
需手动 bind
默认指向组件实例（类组件），或者使用箭头函数

移除事件
需要 removeEventListener
React 自动管理，无需手动移除（组件卸载时自动清理）


示例（React）：

function App() {
  const handleClick = (e) => {
    console.log(e.type); // SyntheticEvent
  };

  return <button onClick={handleClick}>Click me</button>;
}

2️⃣ 条件渲染有哪些方式？返回 null 有什么作用？

常见方式：
	1.	三元运算符

	{isLoggedIn ? <Dashboard /> : <Login />}

	2.	逻辑与 (&&)

	{messages.length > 0 && <MessageList messages={messages} />}

		3.	if / else 在函数内部

		function Greeting({ isLoggedIn }) {
  if (isLoggedIn) return <Dashboard />;
  return <Login />;
}

	4.	立即执行函数 IIFE（少用）

	{(() => {
  if (isLoggedIn) return <Dashboard />;
  return <Login />;
})()}

返回 null 的作用：
	•	React 渲染 null 时 什么都不显示，但不会报错。
	•	常用于条件不满足时不渲染组件。

	function Notification({ message }) {
  if (!message) return null;
  return <div>{message}</div>;
}

3️⃣ 受控组件是什么？有什么优点？

受控组件 (Controlled Component)：
	•	表单元素的值由 React state 控制，而不是 DOM 自己维护。
	•	输入变化时通过 onChange 更新 state。


示例：

function MyInput() {
  const [value, setValue] = React.useState('');

  return (
    <input
      type="text"
      value={value}        // 值受 state 控制
      onChange={e => setValue(e.target.value)} // 更新 state
    />
  );
}

优点：
	1.	表单数据在 React 中可控，方便校验和提交。
	2.	支持即时验证、格式化、限制长度等操作。
	3.	更容易与其它组件共享状态。

4️⃣ 如何处理多个表单输入的 state 更新？

常用方式：
	1.	单个 state 对象管理所有字段

	function Form() {
  const [form, setForm] = React.useState({ username: '', email: '' });

  const handleChange = (e) => {
    const { name, value } = e.target;
    setForm(prev => ({ ...prev, [name]: value }));
  };

  return (
    <>
      <input name="username" value={form.username} onChange={handleChange} />
      <input name="email" value={form.email} onChange={handleChange} />
    </>
  );
}

2.	每个输入单独 useState（字段少时可行）

const [username, setUsername] = React.useState('');
const [email, setEmail] = React.useState('');

✅ 小结：使用对象管理可以 动态添加字段，在大型表单中更方便。
