1️⃣ Dumb / Smart 组件分别做什么？

概念：
	•	Dumb 组件（展示组件 / Presentational Component）
	•	作用：只负责 UI 展示，不处理业务逻辑。
	•	特点：
	•	接收 props 渲染 UI
	•	不维护自己的状态（除了 UI 状态，如 hover、open）
	•	不依赖上下文或全局状态
	•	优点：复用性高，易于测试
	•	示例：

function Button({ label, onClick }) {
  return <button onClick={onClick}>{label}</button>;
}

	•	Smart 组件（容器组件 / Container Component）
	•	作用：负责 数据获取 / 状态管理 / 业务逻辑
	•	特点：
	•	可以有 state 或使用 Redux/Pinia/Vuex
	•	调用 API、处理事件、管理子组件状态
	•	示例：

function ButtonContainer() {
  const [count, setCount] = useState(0);
  return <Button label={count} onClick={() => setCount(c => c + 1)} />;
}


✅ 核心思路：

Dumb 组件 = 展示，Smart 组件 = 管理。Dumb 不直接操作数据，Smart 控制数据流。

2️⃣ 状态应该放在组件的哪个位置？如何拆分？

原则：状态应该放在最小必要的父组件，既能被需要的组件共享，又不会导致不必要的重渲染。
	1.	局部状态 → 放在具体使用它的 Dumb 组件内
	•	例：按钮 hover、展开/收起状态
	2.	共享状态 → 放在最近的公共父组件
	•	例：表单值、分页页码
	3.	全局状态 → 放在 Store（Redux / Pinia / Vuex）
	•	例：用户信息、购物车、登录状态

拆分策略：
	•	Smart 组件管理状态 + Dumb 组件纯展示
	•	避免“状态泄漏”：不要把状态放在过深的子组件里导致父组件必须重渲染

示意：

App
 ├─ Header (Dumb)
 ├─ TodoList (Smart)
 │    ├─ TodoItem (Dumb)
 │    └─ AddTodoForm (Dumb)
 └─ Footer (Dumb)

3️⃣ 子组件如何修改父组件数据？

在 React / Vue 中都是 单向数据流，子组件不能直接改父组件状态。常用方式：
	1.	通过回调函数传递
	•	父组件传函数给子组件，子组件调用

	function Parent() {
  const [value, setValue] = useState(0);
  return <Child onChange={newVal => setValue(newVal)} />;
}

function Child({ onChange }) {
  return <button onClick={() => onChange(42)}>Set 42</button>;
}

	2.	通过全局状态 / Context
	•	Redux / Pinia / Vuex / React Context
	•	子组件直接 dispatch action 或修改 store
	3.	在 Vue 中也可以用 emit

	<Child @update="value = $event" />

	✅ 面试点：绝对不能直接改父组件的 state，只能通过“父给函数 / store”方式间接修改。

4️⃣ 分层组件有什么优势？

分层（Smart / Dumb / UI / Container / Page）组件的好处：
	1.	提高复用性
	•	Dumb 组件只做展示，能在多个页面复用
	2.	降低耦合度
	•	数据逻辑与 UI 分离，便于维护
	3.	便于测试
	•	Dumb 组件可以单独做快照测试 / 单元测试
	4.	可扩展性
	•	Smart 组件管理数据，可自由替换展示组件
	5.	优化性能
	•	减少不必要渲染，状态集中管理

示意：
Page
 └─ Smart Container
      ├─ Dumb UI Component
      └─ Dumb UI Component

	Smart 组件做数据流 + 业务逻辑，Dumb 组件纯展示，分层清晰。
