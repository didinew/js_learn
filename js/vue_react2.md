开场：自我介绍 & 项目概览

面试官一般会快速了解你的背景，顺便判断你经历的项目规模。

问题：

“请介绍你最近负责的项目，并说明你在项目中的角色和主要贡献。”

满分答法模板：
	1.	项目概况（中大型前端项目 / 用户量 / 技术栈）
	2.	自己角色（独立负责模块 / 架构决策 / 核心逻辑）
	3.	主要贡献（组件设计 / 状态管理 / 性能优化 / 工程化）
	4.	亮点（解决复杂业务 / 性能优化 / 模块化 / 高可维护性）

示例：

最近我负责一个中大型电商管理后台，Vue3 + Pinia + TypeScript。
我负责整个商品管理模块架构设计，拆分组件，设计状态管理方案，同时优化列表页渲染性能，提升了列表操作体验。

Round 1：技术选型 + 项目架构

Q1️⃣

“假设你负责一个中大型前端项目（Vue 或 React 都可以），从 0 到 1，你会如何做技术选型和整体架构设计？”

追问：
	1.	为什么选 Vue 或 React？
	2.	状态管理方案如何设计？
	3.	组件拆分原则是什么？

满分答案结构：
	1.	技术选型：根据 团队规模 + 项目复杂度
	•	Vue：快速上手，中小型团队
	•	React：可控性强，大型复杂项目
	2.	项目结构：
	•	分层：UI 层 / 业务逻辑层 / 状态层 / 基础能力层
	•	目录清晰 + ESLint + 约束规则
	3.	组件设计：
	•	功能单一，高内聚低耦合
	•	公共逻辑下沉到 hooks / useXXX
	4.	状态管理：
	•	低频、UI 局部状态组件内管理
	•	高频或跨组件状态使用全局管理（Pinia / Redux）
	5.	性能与可维护：
	•	虚拟 DOM / 缓存 / 组合函数
	•	避免全局状态滥用

⸻

Round 2：状态边界 & 跨组件状态

Q2️⃣

“一个复杂列表页：顶部筛选条件、分页列表、每项 hover/展开/选中，哪些状态放组件内？哪些放全局？”

追问：
	1.	如果选中状态会提交后端，怎么办？
	2.	hover / 展开状态可以全局管理吗？

满分答案：
	•	组件内部状态：局部使用，props / emit 传递
	•	顶部筛选条件、分页列表页码
	•	全局 / 模块化状态：跨组件或后端交互
	•	每项选中状态，跨组件共享或提交后端
	•	原则：局部状态归组件，跨组件状态归全局；保持高内聚、低耦合、可追踪

⸻

Round 3：性能优化

Q3️⃣

“列表页访问量大，用户频繁操作 hover / 展开 / 勾选，你如何保证性能不卡？”

追问：
	1.	大量列表项的渲染优化？
	2.	高频事件如何控制？
	3.	对组件重渲染怎么优化？

满分答案：
	1.	渲染优化：虚拟列表 / windowing，只渲染可视区域
	2.	事件优化：hover / 展开 / 勾选使用节流或防抖
	3.	渲染调度优化：requestAnimationFrame / 批量更新
	4.	可选加分：
	•	useMemo / useCallback / computed 缓存计算
	•	局部 vs 全局状态分层，减少不必要渲染

⸻

Round 4：Vue × React 深层对比

Q4️⃣

“Vue 自动依赖收集 vs React 显式依赖数组，你更倾向哪种？为什么？”

追问：
	1.	Vue 的优势和成本
	2.	React 的优势和成本
	3.	如何在大型项目中选择？

满分答案：
	•	Vue 自动依赖收集：
	•	优点：降低心智负担，快速开发
	•	适用：中小型项目或经验层次参差的团队
	•	React 显式依赖：
	•	优点：可控性强，可预测性高
	•	适用：大型项目，复杂状态，团队有规范
	•	总结：Vue 用机制换简单，React 用显式换可控

⸻

Round 5：架构复用与可维护性

Q5️⃣

“你如何设计一个可长期维护的组件体系？”

追问：
	1.	基础组件 vs 业务组件 vs 页面组件划分
	2.	复用逻辑如何抽取
	3.	避免组件滥用策略

满分答案：
	•	基础组件：无业务逻辑，高复用，props / emit 明确
	•	业务组件：服务单一业务域，逻辑抽离，UI 不跨域复用
	•	页面组件：组合业务组件，生命周期集中处理
	•	原则：不追求一开始完美抽象，出现第二次复用再抽象

⸻

Round 6：真实事故复盘

Q6️⃣

“讲一个你在项目中遇到的线上问题，并分析原因和解决方案”

满分答案结构：
	1.	问题描述（简明、突出痛点）
	2.	根本原因（状态漂移 / 异步更新 / 响应式滥用）
	3.	解决方案（拆分逻辑 / 批量更新 / 防抖节流 / 状态分层）
	4.	收获总结（体现经验和工程判断力）

示例：

列表页 hover + 勾选导致频繁渲染，用户操作卡顿
原因：事件高频 + 全局状态导致多次渲染
方案：虚拟列表 + 防抖 + 状态分层
收获：学会按使用频率区分状态，避免全局状态滥用
